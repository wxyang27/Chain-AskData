# 考察维度框架

面试官按候选人方向选模块加载。每模块含考察维度 + 典型提问方向 + 深挖追问点。

## 一、数据仓库与建模

- 数仓分层：ODS / DWD / DWS / DM / ADS 各层职责、粒度、复用
- 总线矩阵：业务过程 × 维度的构建
- 建模方法论：ER 建模 vs 维度建模，优缺点与选型
- 缓慢变化维（SCD）：重写 / 新增行 / 拉链表 / 前快照
- 主题域划分、事实表（事务/周期快照/累积快照）、维度表、宽表
- 维度退化、缓慢变化维、一致性维度
- 典型追问：
  - "如果让你重构 DWD / DWS，怎么设计？"
  - "你们怎么分层的？DWS 粒度怎么定？"
  - "拉链表怎么处理 SCD？除了拉链表还有什么？"

## 二、数据开发调优实战（面试官杀手锏，重点深挖）

### 2.1 数据倾斜
- 定位：SparkUI stage duration / read 倾斜、key 分布
- groupby 倾斜：加盐打散 + 还原
- join 倾斜：map join / broadcast hash join；大表加盐 + 小表复制 n 份
- 热点隔离 + 局部加盐：采样热点 key → 拆普通/热点 → 分类治理 → 结果合并
- 典型追问：
  - "加盐怎么还原 id？split 打散个数注意什么？"
  - "A 表加盐 n 份，B 表怎么办？为什么小表也要复制 n 份？"
  - "热点 key 怎么治理？讲讲完整方案。"

### 2.2 shuffle
- 宽窄依赖判断、哪些算子触发 shuffle（groupByKey / reduceByKey / join / repartition）
- shuffle 写读流程、stage 划分
- 典型追问："哪些 Spark 算子会 shuffle？""MR 中怎么实现 TopN？"

### 2.3 小文件治理
- 产生原因、影响、解决
- 参数：合并小分区、调整 reducer 个数
- MR 阶段：combiner 预聚合 + reducer/分区个数
- 存量治理：已产生的上万小文件怎么处理
- 典型追问："16450 个存量小文件怎么处理？""动态分区参数怎么配？"

### 2.4 内存与 OOM
- OOM 定位、executor 内存模型、参数调优
- 数据量级 vs 资源配置
- 典型追问："线上任务 OOM 怎么排查？""memory / core 怎么配？"

### 2.5 执行流程与 DAG
- Spark 提交流程：RM / NM / AM / container / driver / job / stage / taskset / executor
- SparkSQL 执行流程：解析→逻辑计划→优化→物理计划→job/stage/task
- DAG 图关注参数：stage duration、read、shuffle、executor
- 典型追问：
  - "Spark SQL 怎么看参数有没有生效？"
  - "environment 了解过吗？"
  - "Spark 如何定位到具体某一段 SQL？"

## 三、SQL 实战

- 窗口函数：lead / lag / row_number / rank / sum over
- 连续登录问题（含允许间隔一天）
- 留存率计算：lag / dateadd / with as
- 行转列 / 列转行：explode / lateral view
- TopN、热门 N
- not exists / not in 区别（NULL 处理）
- 数据膨胀根因（by 字段、join 爆炸）
- 典型追问：
  - "直播间同时在线人数 + 数据倾斜怎么写怎么治？"
  - "连续登录怎么写？允许间隔一天呢？"

## 四、数据治理

- 数据血缘：表级 / 字段级血缘、追踪工具、问题归因
- DQC：规则配置、覆盖率、空值/唯一性/一致性/及时性校验、告警、误报
- SLA：达标率、时效保障、任务优先级、基线
- 元数据管理：技术/业务/操作元数据、数据地图、资产目录
- 数据安全：权限、脱敏、分级分类
- 一致性校验：上游表数 vs 当前同步数
- 典型追问：
  - "DQC 如何对上游表配置？覆盖率 100% 难不难？"
  - "上游 20+ 只同步了 10+，怎么校验一致性？"
  - "数据治理你了解哪些？做过哪些落地工作？"

## 五、数据分析与机器学习

- 统计基础：中心极限定理、假设检验、p 值、显著性
- AB-test：实验设计、指标选取、显著性判断、分流
- 机器学习：决策树 / 随机森林 / XGBoost 原理与对比、集成学习
- 业务 sense：指标体系、口径、归因
- 数据分析 vs 数开 vs 数仓的岗位认知
- 典型追问："数分和数开区别？""AB-test 怎么判断显著？"

## 六、AI + 数据开发（当前主负责方向）

- data agent：架构、多 Agent 协调、执行策略路由、错误追踪、沙箱预检、熔断
- text2sql：SQL 安全防护（注入/权限/白名单）、准确率评估
- RAG：系统流程、检索优化（混合检索/重排/查询改写）、评估（召回率/忠实度）、动态更新、多阶段召回
- MCP：协议核心、通信方式（stdio/SSE/HTTP）、vs Function Calling、vs Skill
- prompt 工程：原则、设计、实践经验
- 多轮对话、长文本生成、幻觉原因与缓解
- A2A 协议、Agent 范式（ReAct/Reflection/ToT）
- 典型追问：
  - "多 Agent 执行策略怎么智能选择和切换？"
  - "NL2SQL 场景下 SQL 安全怎么防护？"
  - "RAG 检索怎么优化？怎么评估？"
  - "MCP 和 Function Calling / Skill 的区别？"

## 八、算法与数据结构（LeetCode Hot100）

面试官的 ACM 经验加持，重点关注 Hot100 高频题，考察思路而非记忆，引导从暴力解逐步优化。

### 数组与双指针
- 两数之和、三数之和、盛最多水的容器、接雨水
- 二分查找：搜索旋转排序数组、在排序数组中查找元素的第一个和最后一个位置
- 典型追问："怎么想到用双指针？""二分边界条件怎么处理？"

### 链表
- 反转链表、环形链表检测、合并两个有序链表、倒数第 K 个节点
- 快手问过"拼接两个有序数组"（归并排序变体）
- 典型追问："快慢指针原理？""递归 vs 迭代反转？"

### 滑动窗口
- 无重复字符的最长子串、找到字符串中所有字母异位词、和为 K 的子数组（前缀和）
- 典型追问："什么时候收缩窗口？""前缀和 + 哈希表怎么优化到 O(n)？"

### 二叉树
- 遍历（前 / 中 / 后 / 层序）、最大深度、对称二叉树、翻转二叉树、最近公共祖先
- 典型追问："递归和迭代各自优缺点？""Morris 遍历了解吗？"

### 动态规划（DP，校招重点）
- 爬楼梯、打家劫舍、最大子数组和、最长递增子序列、最长回文子串
- 背包问题（01 背包、完全背包）、编辑距离
- 典型追问："DP 状态转移怎么推？""怎么优化空间到 O(1)？"

### 堆与 TopK
- 数组中的第 K 个最大元素（堆 vs 快选）、前 K 个高频元素
- 唯品会问过"Python 实现 Top K"
- 典型追问："堆和快选各自复杂度？""堆怎么手写建堆过程？"

### 栈与队列
- 有效的括号、最小栈、用栈实现队列、单调栈（接雨水、每日温度）
- 典型追问："什么时候用单调栈而不是双指针？"

### 回溯与 DFS
- 全排列、子集、组合总和、N 皇后、电话号码的字母组合
- 典型追问："回溯三要素（路径/选择列表/结束条件）怎么找？""去重剪枝怎么做？"

### 手撕题引导策略
- 先确认题目理解（输入/输出/约束），让候选人先说思路再写代码
- 卡壳时从暴力解开始引导，逐步优化（如 O(n²)→O(nlogn)→O(n)）
- 重点关注：边界条件、空值处理、复杂度分析、代码风格
- 典型面试开场："这道题你做过的吧？来，先说说思路，再写一下。"

## 七、软素质与表达

- 自我介绍（1-3 分钟）：表达、简历提炼、时间把控
- 项目表达：STAR 结构、先结论后展开
- 最难问题与解决：问题定位、方案、取舍
- 学习方法、职业规划、岗位选择动机
- 反问业务：业务好奇心
- 典型追问："讲一个项目中最难的问题及怎么解决的？""为什么做这个方向？"

## 提问节奏建议

- 数据开发（实习/校招）：基础概念 30% + 调优实战 25% + SQL 15% + 算法 15% + 软素质 15%

- AI+数据方向：data agent/text2sql/RAG 35% + 数仓/调优 25% + SQL 10% + 算法 10% + 软素质 20%
